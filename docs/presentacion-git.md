# Guía de capturas para GitHub

Esta guía te ayuda a crear una presentación visual clara y atractiva del proyecto en el repositorio.

## Objetivo

Mostrar en pocos segundos qué resuelve la app y cómo se usa.

## Carpeta recomendada

Guarda todas las capturas en `docs/img/`.

Nombres sugeridos:

- `dashboard-general.png`
- `listado-cotizaciones.png`
- `crear-cotizacion.png`
- `detalle-cotizacion.png`
- `pdf-generado.png`
- `bot-telegram.png`

## Orden recomendado en el README

1. Panel principal
2. Listado de cotizaciones
3. Crear cotización
4. Detalle de cotización
5. PDF generado
6. Integración con Telegram

## Guía rápida de estilo

- Resolución sugerida: 1600x900 o 1920x1080.
- Usa la misma escala de zoom en todas las capturas (100 % o 110 %).
- Evita barras del navegador, notificaciones o datos sensibles.
- Deja visible el contexto del flujo: título de página, tabla o formulario principal.
- Si recortas, mantén proporción horizontal para que la galería se vea uniforme.
- Usa datos de ejemplo realistas y bien escritos en español.

## Comandos Git

```bash
git add docs/img README.md docs/presentacion-git.md
git commit -m "Agrega galería visual para presentación del proyecto"
git push
```
